## ğŸ› ï¸ BACKEND SETUP

### Step 1: Navigate to backend directory
```bash
cd backend
```

### Step 2: Create Python virtual environment
```bash
python -m venv venv

# Activate virtual environment
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate
```

### Step 3: Install dependencies
```bash
pip install -r requirements.txt
```

### Step 4: Configure Database

1. Create PostgreSQL database:
```sql
CREATE DATABASE uber_clone;
```

2. Create `.env` file in backend directory:
```env
DATABASE_URL=postgresql://postgres:yourpassword@localhost:5432/uber_clone
SECRET_KEY=your-super-secret-key-change-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
GOOGLE_MAPS_API_KEY=your-google-maps-api-key
STRIPE_SECRET_KEY=your-stripe-key
REDIS_URL=redis://localhost:6379
```

### Step 5: Run the backend server
```bash
python main.py
```

Backend will run on: **http://localhost:8000**
API Documentation: **http://localhost:8000/docs**

## ğŸŒ API ENDPOINTS

### Authentication
- `POST /api/auth/register` - Register new user
- `POST /api/auth/login` - Login
- `POST /api/auth/driver/register` - Register driver with profile

### Rides
- `POST /api/rides/` - Create ride
- `GET /api/rides/` - Get user rides
- `GET /api/rides/available` - Get available rides (drivers)
- `PATCH /api/rides/{id}` - Update ride status
- `DELETE /api/rides/{id}` - Cancel ride
- `POST /api/rides/{id}/rate` - Rate completed ride

### Admin
- `GET /api/admin/stats` - Platform statistics
- `GET /api/admin/users` - Get all users
- `PATCH /api/admin/users/{id}/toggle-active` - Toggle user status

### Intercity
- `GET /api/intercity/cities` - Get cities
- `POST /api/intercity/cities` - Create city (admin)
- `POST /api/intercity/rides` - Book intercity ride
- `GET /api/intercity/rides` - Get intercity rides

### Vacation
- `POST /api/vacation/` - Create vacation booking
- `GET /api/vacation/` - Get vacations
- `GET /api/vacation/loyalty/points` - Get loyalty points

## ğŸ”§ TROUBLESHOOTING

### Database Connection Error
- Verify PostgreSQL is running
- Check DATABASE_URL in .env file
- Ensure database exists

### Frontend Not Loading
- Run `npm install` in frontend directory
- Check if port 5173 is available
- Verify backend is running

### CORS Errors
- Backend CORS is configured for localhost:5173
- Check CORS settings in main.py
